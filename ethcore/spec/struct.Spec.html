Parameters for a block chain; includes both those intrinsic to the design of the chain and those to be interpreted by the active chain engine.

Fields

name

User friendly spec name

engine_name

What engine are we using for this?

nodes

Known nodes on the network in enode format.

engine_params

Parameters concerning operation of the specific engine we're using. Maps the parameter name to an RLP-encoded value.

builtins

Builtin-contracts we would like to see in the chain. (In principle these are just hints for the engine since that has the last word on them.)

parent_hash

The genesis block's parent hash field.

author

The genesis block's author field.

difficulty

The genesis block's difficulty field.

gas_limit

The genesis block's gas limit field.

gas_used

The genesis block's gas used field.

timestamp

The genesis block's timestamp field.

transactions_root

Transactions root of the genesis block. Should be SHA3_NULL_RLP.

receipts_root

Receipts root of the genesis block. Should be SHA3_NULL_RLP.

extra_data

The genesis block's extra data field.

seal_fields

The number of seal fields in the genesis block.

seal_rlp

Each seal field, expressed as RLP, concatenated.
